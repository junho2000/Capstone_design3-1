# 라즈베리파이를 이용한 교통안전 통합 솔루션

## 국민대 2023-1 전자공학 창의설계 경진대회 동상 수상
## 국민대 2023-1 전자공학 창의설계 경진대회 지식재산 분야 동상 수상

## 개요
- 팀소개 : 안녕하세요 저희는 전자공학부 학생들로 이루어진 국민자동차안전솔루션팀입니다. 저희 팀, 국민자동차안전솔루션은 교통안전을 개선하기 위해 구성된 팀입니다. 저희 팀은 이번 캡스톤 설계를 통해 교통안전 통합 솔루션을 만듬으로서 사회 교통안전 향상에 기여하고자 설계주제를 라즈베리파이를 이용한 교통안전 통합 솔루션으로 했습니다.

조장 : 김준호 (프로젝트 전체 일정 및 팀원 역할 분배, 전체적인 코드 리뷰 및 조언 제공, 지그비 통신 구현, 졸음운전 알고리즘 구현, GUI 구현)

조원 : 김희은, 정상욱, 최성환(PPT, 포스터, 관련문서 작성, 아두이노 화재 및 알코올 감지, 설계 지원금 관리, 시제품 제작)

- 개발동기 : 저희 국민자동차안전솔루션 팀은 교통사고로 인한 피해를 최소화하고 교통안전을 강화하기 위한 목적으로 설계를 시작했습니다. 음주운전, 졸음운전으로 인한 교통사고와 자동차 화재는 현대 사회에서 큰 문제로 인식되고 있으며, 그로 인한 인명 피해와 재산 손실이 존재합니다. 따라서 우리는 이러한 문제를 해결하기 위해 혁신적인 접근 방식을 통해 교통안전을 개선하고자 하였습니다.
개발동기 중 첫 번째로는 화재와 같은 급박한 상황에 대한 조기 감지와 대응이 필요하다는 점입니다. 차량 내부에서 발생하는 화재는 매우 빠르게 확산되어 큰 피해를 야기할 수 있습니다. 따라서 저희는 공기질 센서를 이용하여 화재 발생 시 조기에 감지할 수 있는 시스템을 개발하고자 합니다.
두 번째로는 운전자의 안전 상태에 대한 모니터링이 필요하다는 점입니다. 운전 중 졸음운전은 심각한 교통사고의 원인이 될 수 있습니다. 따라서 얼굴 및 눈 인식 카메라를 활용하여 운전자의 졸음 상태를 실시간으로 모니터링하고 경고할 수 있는 시스템을 개발하고자 합니다.
세 번째로는 음주운전 사고를 예방하기 위한 조치가 필요하다는 점입니다. 음주운전은 생명과 안전에 심각한 위협을 가하는 행위로 알려져 있습니다. 따라서 알코올 센서를 활용하여 운전자의 음주 상태를 측정하고 이를 판별하여 적절한 조치를 취할 수 있는 시스템을 개발하고자 합니다.
마지막으로, 사고 발생 시 신속한 대응이 필요하다는 점입니다. 사고가 발생하면 지체 없이 적절한 구조와 조치가 이루어져야 합니다. 따라서 GPS와 스피커를 활용하여 사고 발생 위치를 신속하게 파악하고 경고음을 발생시켜 구조 요청이나 대응을 원활히 할 수 있는 시스템을 개발하고자 합니다.

- 개발필요성 : 기존 시스템은 대부분 음주운전, 졸음운전, 차량화재에 의한 사고발생시 운전자 또는 제3자가 직접 전화로 신고하는 시스템으로 이루어져있습니다. 이러한 시스템은 재난이 아닌 개별 운전자의 사고에 대해서 조기 예측 및 즉각 대응이 불가하고 골든타임을 놓칠 가능성이 있습니다. 또한 사건 발생 시 즉각적인 정보공유가 늦어 출동에 지연 및 혼란을 발생시킬 수 있습니다.
그렇기 때문에 저희 국민자동차안전솔루션팀은 이러한 현재 시스템을 넘어 통합 관리 시스템이 필요하다고 생각했습니다. 저희 조가 제시하는 통합 시스템은 V2X이라는 기술을 사용해 차량내 다양한 iot센서를 활용해 수집된 데이터를 통합적으로 분석하여 개별 운전자의 위험상황 조기 예측 및 대응을 가능하게 해줍니다.

- 개발목표 : 
1. 교통안전의 강화: 우리는 개발하는 솔루션을 통해 교통안전을 강화하고 사고로 인한 피해를 최소화하는 것을 목표로 합니다. 운전자의 안전 상태를 모니터링하고 위험 상황을 빠르게 감지하여 조치할 수 있는 시스템을 개발하여 교통사고 예방에 기여하고자 합니다.
2. 신속한 대응 및 구조 지원: 사고 발생 시에는 신속하고 정확한 대응이 필요합니다. 따라서 우리는 GPS와 스피커를 활용하여 사고 위치를 신속하게 파악하고 관련 기관에 알림을 보낼 수 있는 시스템을 구축하고자 합니다. 이를 통해 구조 작업을 원활히 진행하고 피해를 최소화할 수 있도록 지원합니다.
3. 데이터의 효과적 활용: 우리는 수집한 데이터를 효과적으로 관리하고 분석함으로써 교통안전에 대한 인사이트를 얻고자 합니다. 데이터베이스를 활용하여 사고 발생 정보, 운전자 상태 등의 데이터를 저장하고, GUI를 통해 시각화된 정보를 관리자가 쉽게 파악할 수 있도록 구현하고자 합니다.
4. 사용자 편의성과 안전성 고려: 저희는 사용자의 편의성과 안전을 고려한 솔루션을 개발하고자 합니다. 운전자의 행동을 감지하고 경고하는 기능은 운전자의 안전을 보장하기 위해 신속하고 정확해야 합니다. 또한, GUI를 통해 사용자가 쉽게 정보를 확인하고 조작할 수 있도록 사용자 친화적인 인터페이스를 구현하고자 합니다.
이러한 목표를 달성하기 위해 우리 팀은 지속적인 연구와 개발을 진행하며, 혁신적인 기술과 창의적인 아이디어를 결합하여 교통안전 분야에서의 문제 해결에 최선을 다할 것입니다.

- 기대효과 : 저희 팀은 라즈베리파이와 아두이노를 활용한 교통안전 통합 솔루션을 설계하고 구현하였습니다. 하드웨어 구성은 공기질 센서, 얼굴 및 눈 인식 카메라, 알코올 센서, GPS와 스피커로 이루어져 있으며, 라즈베리파이와 아두이노를 직렬 통신 방식으로 연결하여 제어할 것입니다.
데이터 수집과 관리를 위해 SQL 데이터베이스를 활용하였습니다. 저희가 개발한 하드웨어에서 수집한 데이터를 데이터베이스에 전송하여 저장하고 이를 통해 사건 발생 시간과 차량 번호 등의 정보를 쉽게 확인할 수 있으며 관련 관리자가 이를 통해 효과적인 관리를 수행할 수 있도록 만들 것입니다.
또한, 저희는 통합관리 시스템 GUI를 개발하여 관리자가 데이터베이스에 저장된 정보를 시각적으로 확인할 수 있도록 할 것입니다. GUI에서는 왼쪽 화면을 통해 구체적인 사건 정보를 확인할 수 있으며, 오른쪽 화면에서는 자동차의 GPS 정보를 이용하여 위험 운전자의 위치를 직관적으로 파악할 수 있도록 지도 API를 활용할 것입니다.
이러한 통합관리 시스템은 경찰서 또는 소방서와 같은 관련 기관에서 사건 발생 시 신속한 대응을 할 수 있도록 도움을 줄 것으로 기대됩니다. 관리자는 GUI를 통해 위험 운전자의 위치를 쉽게 확인하고, 데이터베이스에 저장된 사건 발생 시간과 차량 번호를 통해 필요한 조치를 취할 수 있을 것입니다.
이 외에도 저희 솔루션은 확장 가능성을 가지고 있습니다. 추가적인 센서를 연결하거나 기능을 개선함으로써 더 많은 교통안전 요소를 통합할 수 있습니다.

### 1. alcohol_detect.ino

#### 설명
사용장비 : MQ-3 아두이노 알코올 센서 모듈 [SZH-SSBH-045], XB24CZ7WIT-004 지그비 모듈, 아두이노 우노, 아두이노 (Arduino) 호환 XBee 쉴드 XBee Shield 3.0

두 개의 알코올 센서값의 평균이 임계값인 300을 넘으면 알코올을 감지하도록 구성했음.

감지했을 경우 LED와 부저로 운전자에게 경고를 한 후, Zigbee통신으로 서버인 라즈베리파이에 'alcohol'신호 전송함.

```c
#include <SoftwareSerial.h>
...
SoftwareSerial zigbeeSerial(10, 11);  // ZigBee 모듈을 연결할 시리얼 핀 설정, TX, RX을 10번 11번 핀에 할당한다.
...
zigbeeSerial.begin(9600);   // ZigBee 모듈 시리얼 통신 시작 바우드레이트 9600
...
zigbeeSerial.print("alcohol"); // 라즈베리파이 서버에 alcohol이라는 신호를 보냄
```

#### 시행착오 및 문제해결 방법
알코올 센서가 조금 민감하게 반응해서 한번 감지가 되면 여러번 감지가 됐음. 또한 라즈베리파이 서버에서 1초마다 수신 받도록 했기 때문에 코드 마지막에 delay(2000);을 추가해 경고가 계속울리지 않고 정상적으로 라즈베리파이 서버에 수신하도록 했음.


#### 시행착오 및 문제해결 방법
유해가스/공기질 센서를 하루이상 미리 데워놓아야한다고해서 전날 센서를 6시간 정도 써보고 센서를 꺼놓음. 대회 당일 미리 데우지 않고 사용했지만 감지가 잘됐음. 또한 
이 센서를 사용하기 위해서는 밀폐된 공간에 가스를 넣어줘야 수치가 빠르게 올라갔음. 평소에는 11 ~ 13 정도의 수치를 유지하다 1분정도 밀폐된 상자에 가스를 넣어주면 17 ~ 20정도로 올라갔음. 

불꽃 감지 센서는 불꽃이 가까이 있어야 감지됐음. 또한 감지각을 벗어나면 감지가 안됐음.


### 3. drowsy_detect.py
#### 설명
사용장비 : LOGITECH STREAMCAM, 라즈베리파이 8gb ram (rasbian 64bit), 모니터

One Millisecond Face Alignment with an Ensemble of Regression Trees 논문에 나온 방법으로 얼굴에 특징점을 뽑아 눈 위아래에 있는 두점들의 두쌍의 2-norm의 크기를 이용해 눈을 감았을 때, 눈은 반쯤 감았을 때, 눈을 완전히 감았을 때를 감지함. 

sql 관련 설명은 아래 4번 코드에서 설명하도록 하겠음.

- [YouTube 영상](https://www.youtube.com/watch?v=ksi42rwGyas)
- [Github 링크](https://github.com/infoaryan/Driver-Drowsiness-Detection)
이 링크에서 코드를 가져와 졸음을 감지했음. 하지만 동양인의 눈의 크기와 저자의 눈의 크기가 다르기 때문에 비율을 조정해줘야 했음. 아래 코드가 조정된 비율임.

```python
def blinked(a,b,c,d,e,f):
	up = compute(b,d) + compute(c,e)
	down = compute(a,f)
	ratio = up/(2.0*down)

	#Checking if it is blinked
	if(ratio>0.24):
		return 2 #active
	elif(ratio>0.2 and ratio<=0.4):
		return 1 #drowsy 경고
	else:
		return 0 #sleep 경고
...

if y1 + y1 > h//3*2:
  cv2.putText(face_frame, "Head up!", (300,100), cv2.FONT_HERSHEY_SIMPLEX, 1.2, color,3)
```

#### 시행착오 및 문제해결 방법
이 코드를 실행하기 위해선 shape_predictor_68_face_landmarks.dat파일이 같은 경로안에 있어야함.

코드를 실행하면 가끔 라즈베리파이 안에서 돌아가지 못할 때가 있었음
1. 라즈베리파이가 충분히 신호가 강한 인터넷에 연결되지 않았을 경우 sql에 데이터를 넣을 수 없어서 파일이 중간에 멈췄음. 신호가 약하면 화면이 멈췄음.
2. 라즈베리파이가 심하게 과열되었을 경우 정상적으로 작동하던 코드가 갑자기 작동이 안될 때가 있었음.

웹캠을 킨 라즈베리파이에 VNC를 이용해 접근할려 했지만 계속 검은 화면이 나왔음. 이 문제를 해결 할 수 없었기 때문에 따로 모니터를 사용했음.


### 4. rasp_zigbee_coordinator.py
#### 설명
사용장비 : XB24CZ7WIT-004 지그비 모듈, 라즈베리파이 4gb ram (rasbian 32bit)

이 코드는 위에 1,2 번 코드로부터 지그비 통신으로 fire, alcohol 신호를 받으면 sql 서버에 올림. 이때 3번 코드는 라즈베리파이에 웹캠이 연결되어 형식이므로 직접 sql서버에 데이터를 넣음.

#### a. 지그비 통신
지그비 통신을 사용하기 위해선 먼저 XCTU를 설치해 지그비 모듈마다 따로 설정을 해줘야한다. 각 지그비 모듈을 송신 장치로 쓸건지 아니면 수신 장치로 쓸건지 설정을 할 수 있음. 기타 자세한 사항은 아래 링크를 따라하면 된다. 이때 따로 지그비 어댑터가 필요하지 않다. 만약 라즈베리파이 GPIO핀과 지그비랑 연결할 수 있는 선만 있으면 설정할 수 있음.
- [YouTube 링크](https://www.youtube.com/watch?v=oWKUBBdtUFU)

아두이노와는 달리 라즈베리파이에서는 기본 tx rx핀이 블루투스 통신으로 고정 할당되어 있으므로 고정을 해제하거나 다른 핀을 tx, rx핀으로 사용해야했음. 그래서 uart3을 활성화해 4번 5번 핀을 tx, rx 핀으로 활용했음. 아래는 참고한 블로그 글임. 만약 수신이 제대로 안된다면 tx, rx핀을 반대로 꼽았을 가능성이 매우 큼.
- [블로그 링크](https://m.blog.naver.com/emperonics/222039301356)

```python
#지그비 설정
xbee = serial.Serial()
xbee.port = "/dev/ttyAMA1" #uart3을 사용하게되면 /dev/ttyAMA1이 열림
xbee.baudrate = 9600 #아두이노와 같은 바우드레이트 설정
#1초마다 수신
xbee.timeout = 1
xbee.writeTimeout = 1

#신호를 받는 코드, 라즈베리파이에서는 수정했지만 github에선 아직 수정안됨. 
#신호받은 줄에 fire이나 alcohol이 포함되어 있을 경우 sql 테이블에 넣음
data = xbee.readline().strip()

if data == b'fire':
  fire_alert = 1
            
if data == b'alcohol':
  alcohol_alert = 1
  
```

또한 sql에 데이터를 넣을 때 10번 까지 밖에 들어가지 않았음. 이게 데이터를 넣을 때마다 기존 MarkerID를 확인하고 1씩 더해 데이터를 넣어줘야하는데 처음에 구현한 cursor.execute("SELECT MAX(MarkerID) FROM person")코드로는 MarkerID가 10이상 들어가지 않았다. 일의 자리수만 확인하고 넣기 때문에 sql에 Primary Key로 지정한 MarkerID가 같은게 생겨 충돌했음.
